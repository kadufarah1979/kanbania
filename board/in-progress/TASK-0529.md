## Descricao

Executar o fluxo E2E completo no hardware fisico: provisioning WiFi via AP mode,
conexao MQTT, publicacao de leituras de todos os sensores, visualizacao no
dashboard em tempo real e disparo de alertas para parametros fora da faixa.

## Criterios de Aceite

- [ ] ESP32 entra em AP mode e conclui provisioning WiFi -> MQTT com sucesso
- [ ] Metricas de todos os sensores chegam ao TimescaleDB: temp, TDS, pH, salinidade
- [ ] Dashboard exibe leituras em tempo real via WebSocket (sem refresh)
- [ ] Alerta dispara quando parametro configurado sai da faixa (ex: temperatura)
- [ ] Logs MQTT Explorer capturados mostrando topicos e payloads reais
- [ ] Query SQL no TimescaleDB confirmando dados persistidos
- [ ] Evidencias salvas em `docs/hardware/test_results/e2e_full/`
