## Descricao

Auditar todas as strings hardcoded no app e centralizar em arquivos de traducao pt-BR.
Preparar estrutura para suporte a outros idiomas no futuro (mesmo que so pt-BR seja entregue agora).

## Criterios de Aceite

- [ ] Biblioteca: `i18next` + `react-i18next` (ou `expo-localization` + objeto simples)
- [ ] Arquivo `src/shared/i18n/pt-BR.json` com todas as strings
- [ ] Zero strings hardcoded em componentes (exceto emojis e icones)
- [ ] Nomes de parametros internalizados: "Temperatura", "pH", "TDS", "Amonia" etc
- [ ] Formatos de data/hora: `dd/MM/yyyy HH:mm` (padrao brasileiro)
- [ ] Formatacao de moeda: `R$ 1.234,56`

## Notas Tecnicas

- Usar `t('key')` do react-i18next ou constantes centralizadas
- Nao e necessario multi-idioma agora — apenas centralizar strings para facilitar futura traducao
