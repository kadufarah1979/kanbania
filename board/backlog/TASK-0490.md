## Descricao

Corrigir a configuracao do Grafana que permite acesso anonimo com role de Admin
e tem senha de admin configurada como "admin".
Estas configuracoes sao vulnerabilidades de nivel ALTO identificadas na analise PCI.

## Criterios de Aceite

- [ ] `GF_AUTH_ANONYMOUS_ENABLED=false` no docker-compose.yml do Grafana
- [ ] `GF_AUTH_ANONYMOUS_ORG_ROLE` removido ou definido como Viewer (se anonimo for necessario)
- [ ] Senha admin do Grafana alterada para uma senha forte (min 16 chars)
- [ ] Senha forte armazenada no Secrets Manager: `tecpag/prd/grafana`
- [ ] Grafana acessivel somente com autenticacao apos a mudanca
- [ ] Verificar que dashboards existentes continuam funcionando

## Contexto Tecnico

Localizacao: `DevOps/telemetry/docker-compose.yml:145-148`
Configuracoes atuais problematicas:
- `GF_AUTH_ANONYMOUS_ENABLED=true`
- `GF_AUTH_ANONYMOUS_ORG_ROLE=Admin`
- `GF_SECURITY_ADMIN_PASSWORD=admin`
