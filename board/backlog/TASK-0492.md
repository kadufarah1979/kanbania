## Descricao

Migrar a SECRET_KEY fraca e a SignOZ API Key do Frontend (transaction-portal) para o Secrets Manager.
A SECRET_KEY atual e fraca (menos de 256 bits) e deve ser substituida por uma chave segura.

## Criterios de Aceite

- [ ] Nova SECRET_KEY gerada com 256 bits (64 chars hex ou equivalente)
- [ ] SECRET_KEY e SignOZ API Key armazenadas em `tecpag/prd/frontend`
- [ ] Frontend atualizado para buscar secrets do Secrets Manager no startup
- [ ] `.env` do frontend sem valores hardcoded
- [ ] Frontend testado e funcionando
