## Objetivo

Unificar o modulo FinOps (relatorios de custo AWS por OU) e o ConsoleAI operacional (inventario, workflows, aprovacoes) em uma plataforma coesa com autenticacao multi-tenant. Ao final do Q1, cada tenant deve ver seu custo AWS ao lado do inventario de recursos e fluxo de aprovacao — tudo autenticado e isolado por OU.

## Key Results

### KR-01: Auth gate FinOps

- **Meta**: 100 %
- **Atual**: 100 % — entregue na sprint-077 (React frontend com Bearer auth em todas as pages FinOps)
- **Como medir**: % de pages FinOps (Sumario, FinopsTenant, Pendentes, Relatorio, Inventario) com autenticacao Bearer ativa

### KR-02: Visibilidade de custo por tenant

- **Meta**: 100 % de tenants com custo exibido
- **Atual**: 60 % — Dashboard e paginas FinOps funcionais, porem persistencia ainda em SQLite sem cache adequado; migracao PostgreSQL (sprint-078) deve completar este KR
- **Como medir**: verificar no dashboard se todos os tenants ativos da plataforma exibem custo AWS atualizado

### KR-03: Dashboard unificado em producao

- **Meta**: 1 dashboard
- **Atual**: 1 — entregue na sprint-077; React Dashboard com KPIs de custo + inventario + auditoria por tenant, servido via nginx + docker
- **Como medir**: dashboard React acessivel em producao com todos os papeis (DevOps, Gestor)

### KR-04: Performance do dashboard

- **Meta**: 3 segundos (max)
- **Atual**: sem medicao formal — backend sem cache PostgreSQL; meta sera atingida apos sprint-079 (TASK-0569)
- **Como medir**: tempo de carregamento do dashboard React com cache ativo medido em ambiente de staging
