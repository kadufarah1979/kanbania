## Descricao

Configurar o profile de producao no EAS Build (AAB assinado para Play Store),
gerar o primeiro build de producao e criar o draft do listing na Play Store
(titulo, descricao, screenshots, icones, categoria).

## Criterios de Aceite

- [x] `eas.json` com profile `production`: `{ "android": { "buildType": "app-bundle", "credentialsSource": "remote" } }`
- [ ] Keystore configurada via EAS Secrets (nao versionar o .jks) — MANUAL: `eas credentials`
- [ ] `eas build --platform android --profile production` gera AAB sem erros — MANUAL: requer conta EAS
- [ ] App Bundle enviado ao Internal Testing Track no Play Console — MANUAL: requer Play Console
- [x] Listing draft: titulo "AquaBook", descricao PT-BR, categoria "Ferramentas" — documentado em PLAY_STORE_SUBMIT.md
- [ ] Screenshots: pelo menos 2 (HomeScreen + AquariumDetail) — MANUAL: requer dispositivo/emulador
- [x] Icone 512x512px e feature graphic 1024x500px — gerados em `mobile/assets/store/`

## Notas Tecnicas

- Keystore: `eas credentials` para gerar e armazenar
- versionCode deve ser incrementado a cada build
- Screenshots: usar o Z Fold 5 ou gerar com emulador de tamanho padrao (1080x1920)
- `eas.json` atualizado com credentialsSource remote e gradleCommand correto no development
- `app.config.ts` recebeu campo `updates.url` para EAS Updates
- serviceAccountKeyPath corrigido para `../fastlane/google-play-key.json`
- Assets de store gerados em `mobile/assets/store/` (icon-512.png + feature-graphic-1024x500.png)
- Documentacao completa: `docs/dev/PLAY_STORE_SUBMIT.md`
