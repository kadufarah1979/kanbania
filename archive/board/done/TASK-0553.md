## Descricao

Implementar as duas paginas de administracao no React, visiveis somente para usuarios com role=admin: Usuarios & Tenants (CRUD de usuarios e associacao de tenants) e Configuracoes (parametros do sistema, thresholds de budget por tenant).

## Criterios de Aceite

- [ ] Pagina Usuarios & Tenants: tabela de usuarios com botoes de edicao e associacao de tenant
- [ ] Pagina Configuracoes: form para editar thresholds de budget por tenant
- [ ] Ambas as paginas redirecionam para `/` se usuario nao for admin (`role !== 'admin'`)
- [ ] Sidebar oculta as opcoes de admin para usuarios nao-admin (ja implementado no Sidebar.tsx)
- [ ] CRUD de usuarios com endpoints FastAPI (criar, listar, atualizar role)
- [ ] Feedback visual (notificacao Mantine) em sucesso/erro de operacoes

## Contexto Tecnico

- Rotas: `/admin/usuarios`, `/admin/configuracoes`
- Guard de admin no sidebar via `user?.role === 'admin'` (ja implementado)
- Endpoints necessarios: `GET/POST/PATCH /v1/admin/users`, `GET/PATCH /v1/admin/tenants`
- Dados em auth.db (usuarios) e consoleai.db (tenants/budget thresholds)
- Usar `@mantine/core`: Modal, TextInput, Select, Switch para formularios
