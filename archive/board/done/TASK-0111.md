## Descricao

Atualizar as CSS variables em `frontend/src/app/globals.css` para refletir a nova paleta de cores da identidade visual AquaBook. Manter compatibilidade com o sistema de design existente (light/dark themes).

### Paleta Nova (do documento de identidade visual)

**Primaria:**
- Azul Profundo: #0D3B66 (sidebar, elementos escuros)
- Azul Oceano: #1F7AE0 (primary, interactive)
- Verde Aqua: #32C48D (accent, sucesso, vida)

**Secundaria:**
- Azul Claro: #A8D8FF (backgrounds leves)
- Cinza Tech: #4A5568 (textos neutros)
- Branco: #FFFFFF

### O que mudar

1. `--primary` light: de Teal (#0D9488, HSL 177 72% 34%) para Azul Oceano (#1F7AE0, HSL 214 76% 50%)
2. `--primary` dark: de Teal Glow para Azul Oceano claro
3. `--sidebar`: de #1E293B para Azul Profundo (#0D3B66)
4. `--sidebar-accent`: de Teal para Azul Oceano
5. `--status-ok`: manter ou ajustar para Verde Aqua (#32C48D)
6. `--ring`: acompanhar primary
7. Gradiente oficial (Azul Profundo -> Azul Oceano -> Verde Aqua) como classe utilitaria
8. Sensor colors ajustar para coerencia

### Nao mudar

- Estrutura de CSS variables (HSL sem wrapper)
- Status colors (warn, crit, off)
- Layout e spacing

## Revisao QA Codex (2026-02-16T00:04:16-03:00)

- Evidencia objetiva: no tema claro, `--primary: 212 76% 50%` com `--primary-foreground: 0 0% 100%` resulta em contraste aproximado de **4.31:1**, abaixo do WCAG AA (4.5:1) para texto normal.
- Impacto: componentes que usam `bg-primary` + `text-primary-foreground` (ex.: botao padrao) nao atendem o criterio "Contraste WCAG AA mantido em ambos os temas".
