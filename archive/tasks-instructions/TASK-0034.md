## Descrição

Implementar WebSocket hub no backend FastAPI para streaming de dados em tempo real.

Endpoint: `ws://localhost:8000/ws/aquarium/{aquarium_id}?token=xxx`

Eventos emitidos:
- `reading` — nova leitura (sensor ou manual)
- `alert` — alerta disparado
- `device_status` — device online/offline
- `alert_ack` — alerta reconhecido

## Critérios de Aceite

- [ ] Endpoint WebSocket com autenticação via token JWT no query param
- [ ] Clients se inscrevem por aquarium_id
- [ ] Broadcast de reading para todos os clients do aquário
- [ ] Broadcast de alert events
- [ ] Broadcast de device status changes
- [ ] Gerenciamento de conexões (connect/disconnect tracking)
- [ ] Auto-ping para detectar clients desconectados
- [ ] Integração com MQTT consumer (consumer → WS broadcast)
