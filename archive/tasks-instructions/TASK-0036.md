## Descrição

Expandir o alert_service básico (TASK-0022) para um engine completo de alertas com notificações, acknowledge e CRUD de regras.

Arquivo: `backend/app/services/alert_engine.py`

## Critérios de Aceite

- [ ] CRUD completo de alert_rules via API
- [ ] Regras padrão criadas automaticamente no seed do aquário (crit/warn por parâmetro)
- [ ] Avaliação real-time a cada leitura (sensor + manual)
- [ ] Suporte a condições: gt, lt, gte, lte, eq, ne
- [ ] Cooldown por regra (evita spam de alertas)
- [ ] Severity: info, warning, critical
- [ ] Acknowledge de alertas via API (POST /api/v1/alerts/events/{id}/acknowledge)
- [ ] Lista de alertas ativos: GET /api/v1/alerts/events/active
- [ ] Histórico de alertas: GET /api/v1/alerts/events (paginado, filtros)
- [ ] Broadcast via WebSocket ao disparar alerta
