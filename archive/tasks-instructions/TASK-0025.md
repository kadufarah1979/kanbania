## Descrição

Expor o catálogo de parâmetros via API REST (read-only) para uso pelo frontend no wizard de criação e configuração.

Endpoints:
- GET /api/v1/parameters — catálogo completo
- GET /api/v1/parameters/profile/{type}/{subtype} — parâmetros para um perfil específico

## Critérios de Aceite

- [ ] /parameters retorna todos os parâmetros do catálogo
- [ ] /parameters/profile/{type}/{subtype} retorna lista filtrada com is_required flag
- [ ] Response inclui: param_code, label, unit, icon, color, ranges
- [ ] Cache-friendly (dados estáticos, pode ter cache header)
